Ojo técnico importante

Para los compañeros, lo correcto sería decirles:

Los Web Services no deberían hacerse como Razor Pages.
Los servicios deberían hacerse en ASP.NET Core Web API o controladores API.

Las pantallas internas sí pueden hacerse con Razor Pages.

Entonces quedaría así:

Tipo	Tecnología recomendada
Portal público	WordPress
Formulario público	WordPress / plugin propio
Servicios Core	ASP.NET Core Web API en C#
Pantallas internas Core	ASP.NET Core Razor Pages
Base de datos	La misma usada por ambos sistemas
